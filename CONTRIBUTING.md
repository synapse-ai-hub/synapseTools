# Contributing

Thanks for wanting to contribute to `synapse-tools`. Keep it simple:

1. **Branch**: Work in `feature/...`, `fix/...` or `chore/...`. Never push directly to `main`.
2. **Commits**: Small, atomic commits. Messages in imperative: "Add null check for empty column", "Fix typo in phoneme docstring".
3. **Before PR**: Update your branch with `main`, run tests/linter if available.
4. **PR**: Open against `main`, describe what changed and why, add reviewers. Use the PR templates (general, fix, or feature) from `.github/PULL_REQUEST_TEMPLATE/`—GitHub will offer them when you create a PR.

For the full Git workflow (branches, rebase, merge steps), see [OnBoarding/GIT_WORKFLOW.md](./OnBoarding/GIT_WORKFLOW.md).
